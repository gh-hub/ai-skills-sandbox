# ADR-001: Typed API client via openapi-typescript + openapi-fetch, sourced from a static generated spec

## Status
Accepted

## Context
`apps/web` talks to the NestJS API (`apps/api`) via relative `/api/...` fetch calls, proxied by nginx in production. Types are currently hand-written and duplicated by hand in `packages/shared-types`, with no compile-time link between the API's actual request/response shapes and what the client believes they are. `apps/api` already builds an OpenAPI document via `@nestjs/swagger` (`DocumentBuilder` + `SwaggerModule.createDocument` in `src/main.ts`), currently only used to serve interactive docs at `/docs`.

## Decision
- Add a script in `apps/api` that builds the Nest application and calls `SwaggerModule.createDocument` directly (no HTTP server, no listening port, no DB connection required) to write the OpenAPI document to a checked-in `openapi.json`.
- In `apps/web`, run `openapi-typescript` against that file to generate TypeScript types, and use `openapi-fetch` as a thin typed wrapper around `fetch`, configured with a relative base URL (`/api`) matching the existing nginx proxy setup.
- House the generated types + client in `apps/web/lib/api-client`.
- Wrap calls through this client with TanStack Query in the application code — no codegen'd query hooks.

## Consequences
- API and client types can drift out of sync only if the `openapi.json` file isn't regenerated after an API change — regeneration is a manual/CI step, not automatic. A follow-up could wire this into CI as a "spec is stale" check, but that's out of scope here.
- `packages/shared-types` remains for genuinely shared hand-written types; it is not deprecated by this change since the generated client types live only in `apps/web`.
- Because generation reads a static file rather than hitting a live server, local codegen doesn't require running Postgres/the API process — lowers friction for regenerating types during frontend work.
- If a second consumer of the API's typed client ever appears, `apps/web/lib/api-client` would need to move to a shared workspace package — deferred until that's a real need.
